# CHANNELERS app — Claude context

Application monorepo for CHANNELERS. **Design lives in `../docs`** — read `../docs/ARCHITECTURE.md` (system design), `../docs/CLAUDE.md` (decisions), and `../docs/CHANGELOG.md` (what's been built) before changing anything. This file is the code map; ARCHITECTURE.md is the detail.

## Commands
- `pnpm dev` — brain → :8787, stage → :5173. **Works with no API key** (stub seeds + offline oracle/choreo fallback); add `OPENAI_API_KEY` to `.env` for real OpenAI.
- `pnpm -r typecheck` — run before claiming done.
- `pnpm --filter @channelers/stage test` / `pnpm --filter @channelers/brain test` — vitest suites.
- `pnpm -r build && SERVE_STAGE=true pnpm --filter @channelers/brain start` — local prod smoke (single-origin).

## Stack (pnpm + TypeScript monorepo)
- **`apps/brain`** — Show Brain: Fastify + `ws` + OSC + OpenAI SDK. Owns visitor data and all AI calls. Also hosts the visitor **dispatcher** (`src/dispatcher.ts`), the **choreographer** loop (`src/choreo.ts`), oracle **tuning** (`src/tuning.ts`), `/api/tts` (routable fallback chain), and an optional token-gated `/agent` WS endpoint for the **Ableton dial-home bridge** (`src/ableton.ts`, off unless `ABLETON_AGENT_TOKEN` set). For cloud deploys, serves the stage build + `/api` + `/ws` from one origin (`SERVE_STAGE`). See ARCHITECTURE §5.x, §5.6, §7, §12; `DEPLOY.md`.
- **`apps/stage`** — Vite/React; one app, role-based routes. Station screens (`/intake`, `/bodyscan`, `/altar`) gate on **confirm-at-station**, not a typed number (`/intake` & `/altar` via `CalledGate`; `/bodyscan` renders straight from `dispatch.state`). `/intake` is visitor self-confirm; `/altar` is **operator-confirmed on its own surface** (the altar performer admits/repools inline — no `/station` screen); `/bodyscan` is **performer-confirmed** (a guide admits the visitor from `/station/bodyscan`). Routes:
  - `/intake` · `/altar` — visitor stations (survey · pose verify + persona). `/altar` is the altar performer's **self-contained** surface (no dispatcher present at the climax): its standby view ports the `/console` overseer controls — the **altar open/close gate** (`AltarGate` → `POST /api/dispatch/altar`) and a **dual-channel Pluribus broadcast** (countdown → room speakers, then user numbers → performer's in-ear; `out.altar.room`/`out.altar.ear`, own `rate.altar` speed dial) — both shown only while no visitor is admitted (via `CalledGate`'s `extra` slot). When a visitor is called/pending it shows **Confirm arrival + Release** (`CalledGate confirmedBy="operator"`); the operator's "Confirm arrival" admits in **one tap**, running `dispatch.confirm`→`arrive` for a pending occupant (so the altar confirms its own call, not `/dispatch`). Then it **defaults to override** for the pose (`Unlock (override)` is the primary action; the camera pose-match is an opt-in `verify by camera` fallback) before the persona pick. `/intake` uses the CRT skin (`CrtShell`).
  - `/bodyscan` — **controls-free front display** (an input-less TV facing the visitor): retro called-number → a fully-stylized **aura** (no webcam shown — `drawAura`: opaque bg + glowing bones + colorblobs) once admitted. Pose **capture is operator-armed** from `/station/bodyscan` (tapping Capture arms a 3.5s framed-stillness hold), then the kiosk runs an **enroll → confirm** flow: the first hold captures the pose, the visitor must **break it and repeat it** (altar-style match ≥0.9 against the just-captured pose) before it saves — so `enrollPose` fires only after the affirming repeat. `?setup=1` exposes the camera picker for one-time install.
  - `/channel` — performer teleprompter + the operator **Altered-State Console** (oracle tuning dials).
  - `/choreo` — Tier 2 choreography feed (text + in-ear TTS).
  - `/perform` — one-device performer console: `altar` (camera-less) · `channel` · `choreo` · `console` · `dispatch` as tabs, **reusing the standalone components**. All stay mounted (inactive tabs `hidden`) so each child's session/socket/audio survives switching and choreo ear-audio keeps playing. The altar tab is camera-less (`<Altar showCamera={false}>`); standalone `/altar` keeps the camera as an opt-in fallback. The `dispatch` tab embeds the standalone `<Dispatch />` queue board; `/dispatch` stays a standalone route too.
  - `/console` — master overseer (visitors / flow funnel / sessions) · **altar open/close gate** (`AltarGate` → `POST /api/dispatch/altar`) · Pluribus altar-ready broadcast (TTS, single-channel output picker, **speed dial** — `SpeedPicker`/`rate.console`, as on `/choreo` + `/channel`). The altar gate + broadcast moved here off `/dispatch`; both are **also** mirrored on `/altar`'s standby surface (where the broadcast runs **dual-channel**) so the altar performer is self-sufficient once the dispatcher leaves.
  - `/board` — public roster as a bare white-phosphor PC terminal (VT323; every visitor number + current location, `called` = inverse-video now-serving; a waiting-and-unplaced visitor shows `WAITING`/`ON HOLD`, or `ALTAR READY` once they've cleared the pre-altar stations, via the pure `boardRows()`, #24/#18; the Time Offering room shows `STATION A - TIME OFFERING` via its slot) · `/dispatch` — lobby-operator queue board (+ flow strip: altar-ready count, bodyscan idle/blocked; right column lists altar-ready visitors. The altar gate + Pluribus broadcast moved to `/console`) · `/souvenir` — QR takeaway.
  - `/station` (`/station/:station`) — per-station performer arrival-confirm view (bodyscan/paper/offering; altar admits on `/altar` itself, intake self-confirms); passive, calls `arrive`/`repool` (+ `complete` "Done" — **paper's only exit** (manual group station), and the **early release** for the timed `offering` room; the Done affordance gates on whether `onComplete` is wired, i.e. paper or a timed station; + bodyscan **Capture pose** arms the kiosk's stillness-hold, and a **camera picker** switches the kiosk's camera remotely — both via `station.cmd`, kiosk cameras reported through `dispatch.state`).
  - `/feed` — Scan/Shred/Feed, the **paper** group station's spectacle screen (kiosk-less). Two kiosk-less **group stations** exist (`config.dispatcher`): `paper` — **manual checkout** (`groupStations`, no dwell, Done at `/station/paper` is its only exit, #17); and `offering` — the **timed "time offering" room** (`timed.offering.dwellMs`, `OFFERING_DWELL_MS`, default 5 min): auto-releases on its dwell, with a manual early **Done** at `/station/offering`. `offering` has no own screen (performer-confirmed at `/station`); on `/board` it shows `STATION A - TIME OFFERING`. (`waitingroom` was retired in #24; the room is back as the new `offering` id.)

  See ARCHITECTURE.md §3 for the route map and the relevant §5 subsection for each one's internals.
- **`packages/shared`** — zod schemas, the `ShowEvent` + OSC contract, the WS divination protocol, tuning, the survey.
- **`packages/oracles`** — persona library (voices, anti-slop deny-list, prompt builder) + the choreographer prompt builders.
- **`packages/ableton-osc-bridge`** — standalone, decoupled OSC bridge (zero `@channelers/*` imports).

## Conventions
- All TypeScript; computer vision runs in-browser (MediaPipe / ArUco) — no Python sidecar.
- **Loose coupling:** anything outward-facing (Anna's audio, Jeff's visuals) goes through the OSC/WebSocket event bus, never internal coupling. Screen-only streams (dispatch, choreo, tuning) stay *off* the OSC contract.
- Human-in-the-loop; offline-resilient — degrade gracefully on API failure, keep the stub/fallback paths working.
- **OpenAI = gpt-4o** for the intake→seeds transform, the live oracle loop, and the choreographer; stream the live loops. Env: `TRANSFORM_MODEL` / `ORACLE_MODEL` / `CHOREO_MODEL`. Check the current OpenAI reference rather than relying on memory.
- **Brain tests run offline by design:** `apps/brain/test/setup.ts` forces `OPENAI_API_KEY=""` so transform/oracle/choreo take deterministic fallbacks. Tests needing the keyed path `vi.mock("../src/config")`.
