## Stack
pnpm + TypeScript monorepo:
- `apps/brain` — Show Brain: Fastify + `ws` + OSC + OpenAI SDK. Owns visitor data and all AI calls. Also hosts an **optional, off-by-default `/agent` WS endpoint** (`apps/brain/src/ableton.ts`, armed only when `ABLETON_AGENT_TOKEN` is set) that the venue's Ableton **dial-home daemon** connects out to; it consumes `ableton-osc-bridge/host` (`createBridgeHost`) and exposes the typed Ableton facade via `getLive()`. Token verified constant-time at the WS upgrade. With no token the Brain runs exactly as before. (Which show events drive Ableton is not yet wired.) For **cloud single-origin** deploys it can also serve the stage build (`SERVE_STAGE=true` → `@fastify/static` + SPA fallback, so one HTTPS origin answers the screens' relative `/api` + `/ws`), and its WS hub sends a **keepalive** ping (`WS_HEARTBEAT_MS`, default 30s) so venue/campus proxies don't reap idle sockets. See `DEPLOY.md`. `/api/tts` (`apps/brain/src/tts.ts`) synthesizes the oracle/choreo voice with a **routable-MP3 fallback chain**: ElevenLabs → OpenAI TTS (`OPENAI_TTS_MODEL`, default `gpt-4o-mini-tts`, neutral voice) → 204 (browser speechSynthesis) — so per-device output routing keeps working without an ElevenLabs key.
- `apps/stage` — Vite/React; one app, role-based routes:
  - `/intake`, `/bodyscan`, `/altar` — station screens now gate on **confirm-at-station** (`CalledGate`), not typing a number: each screen binds an addressable kiosk slot via `useStationPresence` (sends `station.hello { station, kioskId, slotHint? }`; `kioskId` from `?kiosk=` else a stable `localStorage` UUID), idles until a visitor is `called` to its slot, shows the number + **Confirm arrival** (`POST /api/dispatch/arrive`), then runs the existing station work (survey / pose-enroll / verify+persona). `NumberGate` is retired from the stations. **`/intake` is reskinned as a DMV-purgatory CRT terminal** — `CalledGate skin="crt"` shows an amber segmented-LED number + an **I AM** confirm inside a reusable `CrtShell` (vaporwave void · `feTurbulence` static · toggleable scanline/curvature FX, `?crt=off`), and the survey renders as a Windows-9x dialog. The CRT pieces (`CrtShell`, `SegmentNumber`, `useCrtFx`, `styles/crt.css`) are reusable; `/bodyscan` + `/altar` keep the default gate. `/bodyscan` + `/altar` also have a per-tab **camera picker** (`useDevices("videoinput", …)` → `usePoseLandmarker(onFrame, deviceId)`; `?cam=` override) so an iPhone via Continuity Camera can drive the pose webcam. Stage now has a **vitest** suite (`pnpm --filter @channelers/stage test`).
  - `/channel` — performer page: lobby of oracle-ready visitors → teleprompter (renamed from `/station`). Hosts the **Altered-State Console** (`AlteredStateConsole`, operator-only) — live generation dials (preset/sampling/effects/text-pipeline/scope) from the PHARMAICY "Ayahuasca" module; edits a **global** `OracleTuning` over `tuning.set`/`tuning.state` WS messages (off the OSC contract). Brain side: `apps/brain/src/tuning.ts` (`getTuning`/`registerTuning`); pure logic in `packages/shared/src/tuning.ts` (`PRESETS` verbatim, `resolveSampling`, `mangleOutput`, `buildDriftDirective`); applied in `divination.ts` + `transform.ts`. Also hosts a per-tab **earpiece output picker** (`useDevices("audiooutput", …)` → `speak(text, { sinkId })`; `?out=` override) routing oracle TTS to a chosen device for the central-machine + Scarlett 8i6 per-performer in-ear setup, and a per-tab **mic input picker** (`useDevices("audioinput", …)`; `?in=` override) feeding `createRecognizer`'s `{ getDeviceId }` so each window's STT records from a chosen mic (Scarlett input / lav) instead of the system default.
  - `/choreo` — **Tier 2 choreography feed** (text + in-ear TTS): the live movement cue (teleprompter) + a rolling log, fed by `choreo.delta`/`choreo.done` WS messages. Hosts the operator's **react to oracle reply** timing toggle (`GET`/`POST /api/choreo/config`). Brain side: `apps/brain/src/choreo.ts` (`generateFirstPass` at persona-set, `streamCue` per-turn, live `reactToOracle` flag); the per-turn fan-out lives in `divination.ts` `say()`; prompts in `packages/oracles/src/choreographer.ts`. The `choreo.*` channel is screens-only, off the OSC contract (like tuning/dispatcher). **Cues are spoken** (TTS on by default via a "speak cues" toggle, neutral voice) and routed through the same per-tab **output picker** as `/channel` (`speak(text, { sinkId })`) — point it at a dancer's Scarlett feed or a loudspeaker. Because the `choreo.*` channel is broadcast to every screen tagged by `sessionId`, concurrent altar sessions are kept from "reading multiple things at once" by a pure **`lib/choreoFeed.ts`** reducer — per-session delta buffers + one **focused session** owning the single teleprompter at a time (others accumulate silently / log-only until it releases on `done`) — and by `speak()`'s **generation guard** (`lib/speech.ts`), where a newer cue preempts any in-flight TTS instead of playing over it.
  - `/console` — **master overseer** (3 panels): visitors+inline controls / flow funnel reading the **slot array** + per-slot LEDs / active sessions+event log. Keeps a hidden **manual override** (type a number + station → `POST /api/checkin`) as the operator safety net.
  - `/board` — public call display: `#N → STATION`, derived from `dispatch.state` slots in the `called` phase
  - `/dispatch` — lobby-operator **3-zone board** (spec §6): waiting pool (left) · the addressable slots as a responsive grid with online LEDs, the pending **Confirm call**, and re-pool (center) · completed (right). Register arrivals from the header.
  - `/souvenir` — QR takeaway
  - `/feed` — **Scan/Shred/Feed** station (the first **timed group station**, spec `2026-06-22-group-stations-design.md`). Kiosk-less spectacle screen: a webcam over the slot + a physical button (Space/Enter keypress = a USB arcade button/footswitch) grabs a frame → `api.feedPaper` (`POST /api/paper/feed`, data-URL) → Brain `gpt-4o` vision OCR (`apps/brain/src/paper.ts`, offline → placeholder) → emits **`paper.fed { text, fedAt }`** (no `visitorId` — identity-agnostic) on the bus **+ OSC** (`/channelers/paper/fed`); the route animates the fed text "into the matrix" off that event — `components/FeedMatrix.tsx` drives a fit-to-screen monospace cell grid where the whole text appears readable at once, holds, then quickly transforms into constantly-flipping `0/1` (a fast per-cell ripple), then the field fades out. Shred is physical-only (unmodeled). Pure logic in `lib/paperFeed.ts` (`paperFedText`, `captureDataUrl`) + `lib/paperAnim.ts` (the deterministic, tested animation timeline + `DEFAULT_KNOBS`); styles in `styles/feed.css`.
  - `/waiting` — **deferred** (waiting-room self-serve kiosk, not yet built)
  - The **dispatcher** lives in `apps/brain/src/dispatcher.ts` (`createDispatcher(bus)`) — an **addressable, kiosk-bound slot** engine: slot ids `${station}-${i}` from `config.dispatcher.slots`, capacity = free online slots, per-slot drop reap, pinned `pending → called → in_progress`. Dispatch state rides the `dispatch.state` WS channel (`slots: Slot[]`, `completed`, `surplus`, `stationsOnline`, `timedDwellMs`) — **never OSC**. Dispatcher logistics are deliberately kept off the `ShowEvent`/OSC contract. **`paper` is a second station *kind* — a timed group station** (`config.dispatcher.timed`): its slots are **always online** (no kiosk), capacity = `config.dispatcher.slots.paper` (group capacity, default 4), eligibility `!paperAt` (non-gating, ungated), and it **completes by a dwell timer from Confirm-call** (`PAPER_DWELL_MS`, default 5 min) rather than a task milestone — occupants stay `called` (so `/board` shows them) until `reconcile()` stamps `paperAt`; no-show/stale are bypassed for timed stations. `/dispatch` shows a live "Ns left" countdown per paper occupant (operator-only).
- `packages/shared` — zod schemas, the `ShowEvent` + OSC contract, the WS divination protocol, the survey.
- `packages/oracles` — persona library (voices, anti-slop deny-list, system-prompt builder) **+ the choreographer prompt builders** (`choreographer.ts`: `CHOREO_CLARITY_INSTRUCTION` + first-pass/live/turn prompts).

Run: `pnpm dev` (brain → :8787, stage → :5173). Works with **no API key** (stub seeds + offline oracle fallback); add `OPENAI_API_KEY` to `.env` for real OpenAI.

Deploy: **cloud single-origin** — the Brain serves the stage build + `/api` + `/ws` from one HTTPS origin, on one always-on Fly machine (the store is in memory: no scale-to-zero, no horizontal scale). `Dockerfile` + `fly.toml` + `DEPLOY.md` live in `app/`. Local prod smoke: `pnpm -r build && SERVE_STAGE=true pnpm --filter @channelers/brain start`.

## Conventions
- All TypeScript; computer vision runs in-browser (MediaPipe / ArUco) — no Python sidecar.
- Loose coupling: anything outward-facing (Anna's audio, Jeff's visuals) goes through the OSC/WebSocket event bus, never internal coupling.
- Human-in-the-loop; offline-resilient (degrade gracefully on API failure — keep the stub/fallback paths working).
- OpenAI: **gpt-4o** for the intake→seeds transform (now **music-only** — `Seeds = { music }`; dead dance/persona seeds removed), the live oracle loop, and the Tier 2 choreographer (stream the live loops; OpenAI caches prompts automatically). Configurable via `TRANSFORM_MODEL` / `ORACLE_MODEL` / `CHOREO_MODEL` env. Check the current OpenAI API reference rather than relying on memory.
- Brain tests run **offline by design**: `apps/brain/test/setup.ts` forces `OPENAI_API_KEY=""` so transform/oracle/choreo take deterministic fallbacks (app/.env has a real key). Tests needing the keyed path `vi.mock("../src/config")`.
- Typecheck before claiming done: `pnpm -r typecheck`.
