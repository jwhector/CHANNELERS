## 2026-06-29 ‚Äî Prompt Lab (`/prompt-lab`): frontend-only oracle iteration tool

- **What:** New stage route `/prompt-lab` for fast workshop iteration on the oracle. Three panels: (1) an **intake composer / override view** ‚Äî renders the 19 `SURVEY` fields by kind (text/longtext/single/multi/scale), with Randomise, Load-current (pre-fill from a seeded visitor's stored survey), and a Raw-JSON escape hatch; (2) a **live system-prompt preview** that calls the shared `buildSystemPrompt(PERSONAS[id], survey)` ‚Äî the *exact* string the brain assembles at `session.start` ‚Äî updating as you edit, and via Vite HMR when `personas.ts`/`buildPrompt.ts` change; (3) an **inline divination** reusing the existing divination WS (`session.start`/`say`/`end`, `oracle.delta`/`done`). "Apply & channel" POSTs the composed intake to the existing `/api/visitors/:id/intake` (overwrites freely) then (re)starts a session; "Regenerate" ends the live session and re-starts so the brain rebuilds the prompt from the just-applied survey. Pure form‚Üî`SurveyResponse` + randomiser helpers in `apps/stage/src/lib/promptLab.ts` (unit-tested).
- **Why:** Show-day ask for a faster loop to test how oracle output shifts as the intake and the prompting change, without risky backend edits. Design locked two scope decisions (spec): **prompt axis = preview-only** (the page never sends an overridden prompt ‚Äî to change prompt *text* you edit the source and the `tsx watch` brain auto-reloads) and **compare style = tweak-and-regenerate** (single-shot, no A/B or matrix). Kept **frontend-only**: reuses `GET /api/visitors`, `POST /api/visitors/:id/intake`, and the divination WS ‚Äî **zero backend / shared changes**, additive route, lowest show-day risk. Re-channelling a previously-ended visitor is intentional and the brain allows it (`start()` checks survey/archetype/not-in-session, not `sessionEndAt`).
- **Files/areas:** `apps/stage/src/routes/PromptLab.tsx` (route + `FieldInput`), `apps/stage/src/lib/promptLab.ts` (+ `.test.ts`, 10 cases), `apps/stage/src/styles/promptlab.css`, `apps/stage/src/App.tsx` (route + Home nav entry), `apps/stage/package.json` (adds `@channelers/oracles` workspace dep ‚Äî first stage consumer of the oracles package, for the preview). Design spec: `docs/superpowers/specs/2026-06-29-prompt-lab-design.md`.
- **Verification:** TDD on the pure lib (10/10); `pnpm -r typecheck` 0 errors (all 5 projects); full stage suite 165 pass ‚Äî the only failure the pre-existing, unrelated `BodyScan.test.tsx` pose-timing test (confirmed identical with these changes stashed at clean HEAD).

## 2026-06-28 ‚Äî Oracle intake prompt: tier the survey into "confessions" vs absurd trivia + force one elegant collision

- **What:** Rewrote how `buildSystemPrompt` feeds the visitor's intake to the oracle. Was: a flat, equal-weight dump of `Name: <ticket#>` + all 19 freeText answers keyed by **raw field id** (`aiFeeling: ‚Ä¶`, `shoeSize: ‚Ä¶`) + an always-empty `phrases` line, capped with "you must use the visitor's information." Now: a **tiered, human-phrased** block ‚Äî a curated set of high-signal **confessions** (`broughtYou`, `anyQuestion`, `dontKnow/knowConfident`, `aiFeeling`, `happyMemory`, `tenderTexture`, `cryFrequency`) rendered first with readable labels, then the remaining **absurdist form trivia** as one ignorable line ‚Äî and a new instruction: *let at least one confession AND at least one incidental detail catch on you, and let the two collide‚Ä¶ don't itemize, never read the form back.* Dropped the vestigial `survey.phrases` rendering (the live `/intake` always submits `phrases: []`). Field id ‚Üí human phrasing lives in a `FIELD_PHRASING` map with a fallback to the survey's own label then the raw id, so a new survey field degrades gracefully (lands in trivia until promoted).
- **Why:** The flat dump made every field equal weight, so the oracle hamfisted everything (name-checking shoe size next to "whether my mother forgives me"), and "use the visitor's information" actively encouraged cramming. Jared wanted elegant single connections; the collaborators want to keep the absurdist texture ‚Äî so the design keeps trivia as garnish and asks for a **collision across both buckets** (the small absurd thing as secretly about the large unspoken one), which is where the resonant divination lives. Prompt-only, in `buildPrompt.ts` ‚Äî no new API calls, no architecture, reversible. Pairs with the single-persona collapse + the trimmed altered-state console (same session).
- **Files/areas:** `packages/oracles/src/buildPrompt.ts` (`CONFESSION_FIELDS`, `FIELD_PHRASING`, `fieldPhrasing`, `renderIntake`; `buildSystemPrompt` now composes `renderIntake`; imports `SURVEY` from shared). New `apps/brain/test/buildPrompt.test.ts` (first test for the prompt builder ‚Äî confessions humanized, trivia kept, collision instruction present, raw ids/old instruction gone).
- **Verification:** TDD red (4/5 fail on the old dump) ‚Üí green (5/5); `pnpm -r typecheck` 0 errors; full brain suite 185 pass, the only 2 failures the pre-existing unrelated `dispatcher.test.ts` no-show timers.

## 2026-06-28 ‚Äî Paper-station OCR reliability: high-detail vision, best-effort prompt, hi-res capture, diagnosability

- **What:** Hardened the `/paper` OCR path (no engine swap ‚Äî still gpt-4o vision). (1) **`detail:"high"`** on the `image_url` (was unset ‚Üí defaulted to `"auto"`, which can downsample the frame to 512px and shred small typed text). (2) **Best-effort prompt** ‚Äî the instruction now says transcribe whatever is legible even if faint/partial/skewed and only return empty for a genuinely textless frame (was "If nothing is legible, return an empty string", which made cautious models bail to the placeholder on borderline frames). (3) **Higher-res capture** ‚Äî `/feed` `getUserMedia` now requests `width/height ideal 3840√ó2160` (soft constraint; the default stream is often only 640√ó480) and the JPEG grab went `0.85 ‚Üí 0.92` (less glyph-edge ringing). (4) **Diagnosability** ‚Äî the `/api/paper/feed` handler now logs the outcome (read vs. empty vs. error, with a preview) instead of swallowing it, and with **`PAPER_DEBUG_DIR`** set it dumps each captured frame + OCR result to disk so an unreliable feed is debuggable by eye (dark/blurry/glary capture vs. empty model read). `paper.ts` split into a pure, unit-tested `buildOcrRequest` + `ocrTextFromResponse` (regression guard on `detail:"high"` and the trim/empty contract).
- **Why:** Rehearsal showed the paper station returning "‚ãØ the page is illegible ‚ãØ" most of the time ‚Äî but `PAPER_FALLBACK_TEXT` was an undifferentiated catch-all for offline / empty-read / thrown-error, with errors silently swallowed, so the failure mode was invisible. Root-cause analysis (webcam in a dark room, single overhead light): the model (gpt-4o, set via `.env TRANSFORM_MODEL`) is capable, so the bottleneck is **capture quality + the API not being told to look closely + a prompt that encourages giving up**. The existing Tesseract.js plan was explicitly rejected for *this* symptom ‚Äî local OCR is *more* fragile on poor webcam frames and would raise the illegible rate (it targets determinism, a different goal). For this spectacle, "something legible beats nothing".
- **Files/areas:** `apps/brain/src/paper.ts` (rewrite: `buildOcrRequest`/`ocrTextFromResponse`/`ocrPage` + `savePaperDebugFrame`), `apps/brain/src/app.ts` (paper handler: outcome logging + gated frame dump), `apps/brain/src/config.ts` (`paper.debugDir`), `apps/stage/src/routes/Feed.tsx` (hi-res `getUserMedia`), `apps/stage/src/lib/paperFeed.ts` (JPEG 0.92), `apps/brain/test/paper.test.ts` (new `buildOcrRequest`/`ocrTextFromResponse` tests).
- **Verification:** TDD red‚Üígreen; `pnpm -r typecheck` 0 errors; brain paper+endpoints suites 32 pass (incl. the existing placeholder integration test); stage paper-related suites 27 pass. (Pre-existing, unrelated suite failures from the uncommitted dispatcher-tuning + `BodyScan.tsx` work streams remain ‚Äî they pass at clean HEAD.)
- **Follow-ups for the team:** the biggest remaining lever is **physical** ‚Äî a jig so every page lands flat/full-frame at the cam's focal distance, plus off-axis/raking light on matte paper to kill glare. Also note a latent footgun: `config.transformModel` *defaults* to `gpt-5-mini` while `.env` overrides to `gpt-4o`; running without that env line silently changes the OCR (and intake-transform) model.
- **Docs touched:** this entry. (No route/stack/convention change; `app/CLAUDE.md` already omits paper OCR from its "OpenAI = gpt-4o" line and the new `PAPER_DEBUG_DIR` is a diagnostic env, not an architectural shift.)

## 2026-06-28 ‚Äî Visitor-store snapshot persistence (opt-in crash/redeploy recovery)

## 2026-06-10 ‚Äî Robust divination-session recovery (refresh-safe + orphan reaper)

Root cause: ephemeral client state held the sole handle to a durable server resource whose lifetime was bound to a command rather than to owner liveness.
