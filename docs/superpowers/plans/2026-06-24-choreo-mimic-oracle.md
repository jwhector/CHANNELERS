# Choreo "mimic the oracle" mode — synchronized channeling for the dancers

**Goal:** Let the dancers' `/choreo` feed periodically replace its movement cues with the oracle's actual voice, so the oracle performer and the dancers all hear the same line to channel at the same time.

**Approach:** The brain owns a per-turn decision — manual toggle OR an "every N turns" cadence — and on a *mimic turn* it suppresses the choreographer and broadcasts a new `choreo.mimic` message carrying the oracle's line + its archetype. The `/choreo` screen replays that line (persona voice) on its own already-configured output device. A small TTS cache in the brain makes the dancers' fetch a byte-identical cache hit, so one ElevenLabs synthesis serves both devices.

**Tech stack:** TypeScript monorepo (pnpm). `apps/brain` (Fastify + `ws`), `apps/stage` (Vite/React), `packages/shared` (zod + WS protocol). vitest throughout.

---

## 1. Why / design

### Problem
Today two independent per-session streams fan out of the brain (both broadcast, tagged by `sessionId`):

| Stream | Generated by | Consumed by | Voiced where |
|---|---|---|---|
| **Oracle** (`oracle.delta`/`oracle.done`) | `streamReply()` — the persona | `/channel` (filtered to `mySessionId`) | performer earpiece, persona voice |
| **Choreo** (`choreo.delta`/`choreo.done`) | `runChoreo()` → `streamCue()` — the choreographer | `/choreo` (focused-session reducer) | dancers' device, **neutral** voice |

So the performer-oracle hears the oracle line (to channel); the dancers hear abstract movement cues. We want a *mimic mode* where the dancers instead hear the oracle's voice — so everyone channels the same line simultaneously — toggled manually and/or on a periodic cadence.

### Scope
- **In:** brain-owned mimic mode (manual + cadence), choreographer suppression on mimic turns, a `choreo.mimic` broadcast, `/choreo` replay (persona voice) + banner + operator controls, a brain TTS cache.
- **Out (YAGNI):** the proposed "audio to all devices during intake" feature. This design is deliberately a *degenerate case* of it (brain says "play this," each screen plays on its own output), so that feature is additive later — not a rewrite. The TTS cache is the one piece we build now that it will also need.
- **Unchanged:** `/channel` and the performer earpiece. We *add* the dancers' device; nothing moves off the oracle.

### Approaches considered
- **A — brain pushes a dedicated `choreo.mimic` message (CHOSEN).** Brain skips the choreographer on a mimic turn and broadcasts `{ sessionId, text, archetype }`; `/choreo` just replays it. Brain owns 100% of the logic; the screen stays dumb (no session/archetype/turn tracking); persona voice rides along for free.
- **B — broadcast a mode flag; screen re-subscribes to `oracle.*`.** Rejected: `oracle.done` carries no archetype (screen would have to track `session.started` to recover the persona voice), and a per-turn cadence *pulse* is awkward to encode as a sustained flag racing the oracle stream.
- **C — reuse `choreo.done` for the oracle line.** Rejected: conflates cues with mimic lines in the log/reducer and loses the persona voice (neutral).

### Audio-production decision
**`/choreo` replays it** (not "`/channel` fans out to a second sink"). Each screen plays on its own situated output. This scales to many devices/machines and matches the future intake feature; concentrating audio on `/channel` cannot span machines. The "reuse the same ElevenLabs clip" goal is met at the brain via a `(text, archetype)` cache rather than by coupling the two pages.

### Decision log
- **Triggers:** both — a sustained manual toggle AND an enable/disable cadence ("every N turns"). Effective: `mimicManual || (cadenceEnabled && turnNumber % N === 0)`.
- **Defaults:** both toggles **off**, `N = 3`. Behaves exactly like today until a switch is flipped.
- **Sync:** brain-owned mode (the cadence is a server timer; can't live in a browser tab). Works identically for one screen or many.
- **Choreo cues on a mimic turn:** **fully suppressed** (no choreographer call).
- **Voice:** persona voice (archetype shipped in `choreo.mimic`) so all channelers hear the identical voice.
- **Display:** oracle line on the teleprompter + a "▶ channelling — mimic the voice" banner.
- **TTS cache:** added now — identical clip across devices, one synthesis, and pre-builds the intake feature.
- **`speak cues` toggle** governs *all* `/choreo` voicing (cues and mimic) — one predictable mute.

```ts
// Small LRU so a line voiced for one device (e.g. the performer earpiece) is replayed byte-for-byte
// for others (the dancers' /choreo device) without a second ElevenLabs/OpenAI synthesis. Also the
// primitive the future "one clip → all station devices" intake feature will reuse.
const TTS_CACHE_MAX = 8;
const ttsCache = new Map<string, Buffer>();

/** Test seam: drop the cache so mocked-key tests don't leak synthesized clips across cases. */
export function clearTtsCache(): void {
  ttsCache.clear();
}

export async function synthesizeSpeech(text: string, archetype: string): Promise<Buffer | null> {
  const key = `${archetype} ${text}`;
  const cached = ttsCache.get(key);
  if (cached) return cached;
  const mp3 = await synthesizeUncached(text, archetype);
  if (mp3) {
    ttsCache.set(key, mp3);
    if (ttsCache.size > TTS_CACHE_MAX) ttsCache.delete(ttsCache.keys().next().value as string);
  }
  return mp3;
}
```
